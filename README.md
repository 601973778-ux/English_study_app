# English Study App

基于开源词表与词典数据，面向「背单词 + 智能体陪练」的英语学习应用（规划中）。

## 项目愿景

把**结构化词库**与**大语言模型对话**结合起来：用户既能按计划复习、测验单词，又能通过智能体用自然语言提问——例如要例句、辨析近义词、讲解词根文化背景，或根据学习进度调整难度与节奏。

## 与现有资源的关系

仓库内包含 [DictionaryByGPT4](https://github.com/ceelog/DictionaryByGPT4) 相关资源（见 `DictionaryByGPT4-main/`），提供：

- 单词释义、例句、词根词缀等风格的结构化数据（如 `gptwords.json`）
- 可参考的网页展示与离线词典形态

本应用将在此基础上扩展：**背诵流程**（列表、卡片、间隔复习）、**学习记录**，以及**可配置的智能体**（系统提示词、工具调用、多轮对话），而不是简单复刻静态词典页。

## 功能方向（草案）

| 模块 | 说明 |
|------|------|
| 词书与进度 | 选词书 / 自定义词表；已学、待复习、生词本 |
| 背诵与测验 | 卡片翻转、拼写/选择/听写等模式；可对接间隔重复算法 |
| 学习智能体 | 解释单词、生成例句、纠错、口语/写作引导；遵守安全与适龄内容策略 |
| 账户与同步 | 后期可选：多端同步、匿名本地优先 |

具体交互形态（Web / 桌面 / 移动端）与后端技术栈将在原型阶段确定。

## 本地预览词典子项目

若需单独查看 `DictionaryByGPT4-main` 的静态页面，可在该目录下用本地静态服务器打开 `index.html`（避免 `file://` 跨域问题），或参考其子目录内的 `README.md` 与在线演示链接。

## 路线图（初步）

1. **M0**：理清数据格式，从 `gptwords.json` 等来源抽取最小可用词表与字段规范  
2. **M1**：最小背单词闭环（选词 → 复习 → 标记掌握）  
3. **M2**：接入智能体 API，固定「教师」角色与任务模板，支持单轮/多轮对话  
4. **M3**：进度持久化、简单统计与可配置词书
